## The 8 Dimensions of Platform PM

Platform PM is fundamentally different from application PM. Two objectives: **acceleration** (help others move faster) and **extended value** (enable an ecosystem on your platform).

### 1. Empower the mid-tier, not just end users
Platform PMs optimize for developers, producers, and partners — not direct end users. The multiplier effect: your ecosystem should be larger and more powerful than your company itself.

### 2. Protect core interactions, enable the rest
Identify what drives recurring value — that's your core interaction and competitive differentiation. Protect it. Only after the core is stable should you empower the ecosystem to build on top.
> "Exposing the core creates competitors. Doing it right creates complementary experiences."

### 3. API strategy: private → partner → public
- Sensitive/private data → Private API
- Controlled experience → Partner API
- Unique differentiator → Public API
- Otherwise → consider open source
Most public APIs start internal, then beta with select partners, then go public. APIs must be treated as products before going public.

### 4. Influence microservices architecture
Platform PMs need enough technical depth to shape architecture decisions. Map dependencies between services. Each user story may span multiple microservices.
Key metrics: availability SLAs (99.9x%), API calls/day, read/write latency, events/queries per second.

### 5. Long-term sustainability > short-term wins
Platform strategy spans years. Prioritization must weigh long-term performance, scalability, and API extensibility — not just quick wins. Frame long-term investments through TAM expansion and product line growth.

### 6. Future-proof platform updates
Backward compatibility is non-negotiable. Developers build businesses on your APIs. Support older versions until partners have migrated. Breaking changes without a migration path will disengage your ecosystem.

### 7. Platform monetization models
- Revenue share: % of partner revenues
- API subscription: tiered by volume or features
- API call volume: pay per use
- Premium access: elevated/priority access
- Data monetization: licensing insights
- Indirect: use platform to enhance core product experiences, drive loyalty/acquisition

### 8. Platform-first culture
Platform requires autonomous, self-organized teams with high alignment. Platform PMs lead through influence and business case, not backlog authority. Allow time for "spiking" — architecture research before sprints.
Culture traits: servant leadership, empowerment and trust, continuous improvement, iterative/agile development.
